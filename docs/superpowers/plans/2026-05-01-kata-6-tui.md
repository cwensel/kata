**Goal:** Add `kata tui`, a Bubble Tea terminal UI for browsing and mutating issues. List view (default), Detail view (header + body + comments|events|links tabs), Help view. SSE-driven invalidation (refetch on event, no client-side replication). Inline create + `$EDITOR` suspend/resume for body and comment edits. Color-mode aware (KATA_COLOR_MODE / NO_COLOR / Lipgloss adaptive). Cross-project mode via `--all-projects` / runtime `R` toggle.

**Architecture:** A Bubble Tea TEA loop sibling to existing CLI verbs. The TUI talks to the local daemon over the same HTTP API the CLI uses (no SQLite bypass). State lives in a top-level `Model` that dispatches `Update`/`View` to one of three sub-views. A long-lived goroutine reads `/api/v1/events/stream` and forwards SSE frames into the TEA event loop as `eventReceivedMsg`; the TUI marks the active query stale and schedules a 150ms-debounced refetch. `sync.reset_required` drops cache, refetches, and shows a 2s "resynced" toast. `$EDITOR` workflows use `tea.ExecProcess` so the renderer correctly suspends and resumes around the child process.

## Architecture

### TEA model shape

```go
// internal/tui/model.go (sketch)
type viewID int

const (
    viewList viewID = iota
    viewDetail
    viewHelp
    viewEmpty // no project bound and no projects registered
)

type Model struct {
    opts        Options       // immutable: includeDeleted, allProjects, daemon endpoint, color mode
    api         *Client       // typed daemon client
    
    view        viewID
    list        listModel
    detail      detailModel
    help        helpModel
    
    width       int
    height      int
    
    // Async machinery:
    sseCh       chan sseFrame
    sseStop     chan struct{}
    pendingRefetch bool         // set when a 150ms debounce is in flight
    
    // Status surface:
    toast       toast          // transient (~2s) message ("resynced", "comment added", error text)
    sseStatus   sseConnState   // connected | reconnecting | disconnected
    
    // Project context (resolved at boot):
    project     *resolvedProject // nil when allProjects==true
    projects    []projectSummary // populated when allProjects==true
}
```

### Update dispatch

`Update(msg)` always handles a few global messages first (`tea.WindowSizeMsg`, `tea.KeyMsg` for `?`/`q`, `eventReceivedMsg`, `tickMsg` for toast expiry), then delegates to the active sub-model's `Update`. Sub-models return their next state plus a `tea.Cmd`. The top-level dispatch never inspects a sub-model's internal state directly.

### Cache + invalidation

The list view holds a `*issueCache` keyed by `(scope, statusFilter, ownerFilter, labelFilters, search, includeDeleted)`. Each cache entry has `data []Issue, fetchedAt time.Time, stale bool`. SSE events mark the active key stale; a 150ms-debounced `refetchTickMsg` then refetches and replaces the entry. Detail view holds a single `currentIssue *Issue` pointer; an SSE event matching the open issue's `(project_id, issue_number)` triggers an immediate single-issue refetch.

### SSE goroutine

`startSSE(ctx, endpoint, projectID *int64, sseCh chan<- sseFrame)` runs in a goroutine. Standard SSE parsing:
- Lines starting with `:` are comments (heartbeats — ignored).
- `id: <n>\nevent: <type>\ndata: <json>\n\n` blocks are event frames.
- On `event: sync.reset_required`, emits a `frame{kind: resetRequired, afterID: data.reset_after_id}`.
- On any other event, emits `frame{kind: event, raw: data}`.
- On read error, the goroutine sleeps with exponential backoff (1s → 2s → 4s → ... → 30s) and reconnects, replaying from the last seen `id` via `Last-Event-ID` header. The TUI tracks `sseStatus` so the status bar can show "daemon: reconnecting…".

### `$EDITOR` suspend/resume

`tea.ExecProcess(cmd, callback)` is Bubble Tea's primitive: it tears down the renderer, runs the child process attached to the controlling terminal, then re-establishes the renderer on exit. The TUI:

1. Writes a template (e.g. existing body for `e`, empty for `c`/`n`) to `os.CreateTemp("", "kata-*.md")`.
2. Builds `exec.Command($EDITOR, tmpfile)` (shell-quoting handled by `shlex` if `$EDITOR` contains spaces).
3. Wraps in `tea.ExecProcess`. Callback reads the tmpfile, removes it, dispatches `editorReturnedMsg{kind, content, err}`.
4. The model handles `editorReturnedMsg` by submitting the content via the API client and popping a toast on success/failure.

`$EDITOR` defaults to `vi`. The TUI honors the user's `EDITOR` env first, then `VISUAL`, then falls back. Empty content cancels the operation (no API call).

### Color resolution

`theme.go::resolveColorMode()` reads `NO_COLOR` first (highest precedence: any non-empty value strips colors entirely), then `KATA_COLOR_MODE` (`auto` | `dark` | `light` | `none`). `auto` defers to Lipgloss's terminal detection. The single theme uses `lipgloss.AdaptiveColor` so light/dark resolve at render time when `auto`. All styles are package-level vars rebuilt once at boot via `applyColorMode(mode)`.

### Test strategy

Three layers:

1. **Unit** — SSE parser, cache, keymap-vs-help consistency, color resolution. No teatest needed.
2. **TEA snapshots** — `teatest.NewTestModel` renders the model with synthetic API responses (mock `Client` interface). Drive keystrokes (`tm.Send(tea.KeyMsg{...})`), then `tm.WaitFor(...)` and assert on the rendered frame via `tm.Output()`. Golden files in `internal/tui/testdata/`.
3. **Integration** (optional, deferred) — a real ephemeral daemon spawned via `internal/testenv` (the same harness Plan 5 uses for `e2e/hooks_test.go`). One smoke test that boots TUI against a live daemon, fires an issue mutation via the daemon's HTTP API, and asserts the SSE-triggered refetch landed within 1s. Build-tagged `//go:build !windows` to dodge the same socket-path constraint Plan 5 hit.

```go
package tui

import (
	"context"
	"errors"
	"io"
	"os"

	tea "github.com/charmbracelet/bubbletea"
)

// Options controls TUI behavior. Stable across versions; new fields
// must be optional.
type Options struct {
	AllProjects    bool
	IncludeDeleted bool
	Stdout         io.Writer // typically os.Stdout
	Stderr         io.Writer // typically os.Stderr
}

// Run starts the TUI. Blocks until the user quits or ctx is cancelled.
// Returns nil on clean exit. Returns errNotATTY when stdin/stdout are
// not a terminal so callers can print a friendly message.
func Run(ctx context.Context, opts Options) error {
	if !isTerminal(os.Stdin) || !isTerminal(os.Stdout) {
		return errNotATTY
	}
	m := initialModel(opts)
	p := tea.NewProgram(m,
		tea.WithContext(ctx),
		tea.WithAltScreen(),
		tea.WithMouseAllMotion(), // future-proof; ignored by current handlers
	)
	if _, err := p.Run(); err != nil {
		return err
	}
	return nil
}

// errNotATTY indicates the TUI was launched outside a terminal.
var errNotATTY = errors.New("kata tui requires a terminal (stdin/stdout must be a tty)")

func isTerminal(f *os.File) bool {
	fi, err := f.Stat()
	if err != nil {
		return false
	}
	return (fi.Mode() & os.ModeCharDevice) != 0
}
```

```go
package tui

import (
	tea "github.com/charmbracelet/bubbletea"
)

type viewID int

const (
	viewList viewID = iota
	viewDetail
	viewHelp
	viewEmpty
)

// Model is the top-level Bubble Tea model. Sub-views are embedded by
// value so Update can mutate them in place without indirection.
type Model struct {
	opts   Options
	view   viewID
	width  int
	height int
}

func initialModel(opts Options) Model {
	return Model{opts: opts, view: viewList}
}

func (m Model) Init() tea.Cmd {
	return nil
}

func (m Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
	switch msg := msg.(type) {
	case tea.WindowSizeMsg:
		m.width, m.height = msg.Width, msg.Height
		return m, nil
	case tea.KeyMsg:
		if msg.String() == "q" || msg.String() == "ctrl+c" {
			return m, tea.Quit
		}
	}
	return m, nil
}

func (m Model) View() string {
	return "kata tui — Plan 6 scaffolding (press q to quit)"
}
```

```go
package tui

import (
	"os"
	"strings"

	"github.com/charmbracelet/lipgloss"
)

type colorMode int

const (
	colorAuto colorMode = iota
	colorDark
	colorLight
	colorNone
)

// resolveColorMode honors NO_COLOR (any non-empty value) over
// KATA_COLOR_MODE. Unrecognized values fall back to auto.
func resolveColorMode() colorMode {
	if v := os.Getenv("NO_COLOR"); v != "" {
		return colorNone
	}
	switch strings.ToLower(os.Getenv("KATA_COLOR_MODE")) {
	case "dark":
		return colorDark
	case "light":
		return colorLight
	case "none":
		return colorNone
	default:
		return colorAuto
	}
}

// Style vars are package-level so View() functions don't reach into
// state. applyColorMode rebuilds them once at boot.
var (
	titleStyle    lipgloss.Style
	subtleStyle   lipgloss.Style
	statusStyle   lipgloss.Style
	selectedStyle lipgloss.Style
	openStyle     lipgloss.Style
	closedStyle   lipgloss.Style
	deletedStyle  lipgloss.Style
	helpKeyStyle  lipgloss.Style
	helpDescStyle lipgloss.Style
	errorStyle    lipgloss.Style
	toastStyle    lipgloss.Style
	chipStyle     lipgloss.Style
	chipActive    lipgloss.Style
	tabActive     lipgloss.Style
	tabInactive   lipgloss.Style
)

// applyColorMode rebuilds all package-level styles. Called at TUI boot
// so tests can swap modes without leaking state across tests.
func applyColorMode(m colorMode) {
	if m == colorNone {
		// Plain mode: keep bold/italic for emphasis but strip every color.
		base := lipgloss.NewStyle()
		titleStyle = base.Bold(true)
		subtleStyle = base
		statusStyle = base
		selectedStyle = base.Reverse(true) // selection still distinguishable
		openStyle = base
		closedStyle = base
		deletedStyle = base.Faint(true)
		helpKeyStyle = base.Bold(true)
		helpDescStyle = base
		errorStyle = base.Bold(true)
```

```go
// doWithHeaders is do() plus optional request headers. Used by methods
// that need Idempotency-Key or X-Kata-Confirm. The TUI doesn't issue
// destructive ops (delete/purge are CLI-only per §7.3), so X-Kata-Confirm
// is unused for now but the helper is kept for symmetry.
func (c *Client) doWithHeaders(ctx context.Context, method, path string, body, out any, headers map[string]string) error {
	var rdr *bytes.Reader
	if body != nil {
		b, err := json.Marshal(body)
		if err != nil {
			return err
		}
		rdr = bytes.NewReader(b)
	}
	var req *http.Request
	var err error
	if rdr != nil {
		req, err = http.NewRequestWithContext(ctx, method, c.base+path, rdr)
		req.Header.Set("Content-Type", "application/json")
	} else {
		req, err = http.NewRequestWithContext(ctx, method, c.base+path, nil)
	}
	if err != nil {
		return err
	}
	for k, v := range headers {
		req.Header.Set(k, v)
	}
	resp, err := c.hc.Do(req)
	if err != nil {
		return fmt.Errorf("%s %s: %w", method, path, err)
	}
	defer resp.Body.Close()
	if resp.StatusCode >= 400 {
		return decodeError(resp, method, path)
	}
	if out == nil {
		return nil
	}
	return json.NewDecoder(resp.Body).Decode(out)
}

// do issues an HTTP request and decodes JSON. On non-2xx it parses the
// daemon's error envelope and returns it as an error containing code +
// message + hint, so toast messages stay actionable.
func (c *Client) do(ctx context.Context, method, path string, body any, out any) error {
	var rdr *bytes.Reader
	if body != nil {
		b, err := json.Marshal(body)
		if err != nil {

		}

	if rdr != nil {
		req, err = http.NewRequestWithContext(ctx, method, c.base+path, rdr)
		req.Header.Set("Content-Type", "application/json")
	} else {
		req, err = http.NewRequestWithContext(ctx, method, c.base+path, nil)
	}
	if err != nil {
		return err
	}
	resp, err := c.hc.Do(req)

	}
	return json.NewDecoder(resp.Body).Decode(out)
}

// decodeError returns an *APIError containing the daemon's error
// envelope so toasts can render the hint.
func decodeError(resp *http.Response, method, path string) error {
	var env struct {
		Status int `json:"status"`
		Error  struct {
			Code    string         `json:"code"`
			Message string         `json:"message"`
			Hint    string         `json:"hint"`
			Data    map[string]any `json:"data"`
		} `json:"error"`
	}
	_ = json.NewDecoder(resp.Body).Decode(&env)
	return &APIError{
		Method:  method,
		Path:    path,
		Status:  resp.StatusCode,
		Code:    env.Error.Code,
		Message: env.Error.Message,
		Hint:    env.Error.Hint,
	}
}

// APIError is the structured form of the daemon's error envelope. The
// TUI inspects Code in some hot paths (project_not_initialized fallthrough,
// duplicate_candidates display).

	Method, Path string
	Status       int
	Code         string
	Message      string
	Hint         string
}

func (e *APIError) Error() string {
	parts := []string{e.Code, e.Message}
	if e.Hint != "" {
		parts = append(parts, "hint: "+e.Hint)
	}
	return strings.Join(parts, ": ")
}
```

```go
// command. Top-level Update keeps responsibility for global keys (?, q,
// R) and SSE messages.
func (lm listModel) Update(msg tea.Msg) (listModel, tea.Cmd) {
	switch msg := msg.(type) {
	case tea.KeyMsg:
		if lm.search.inputting {
			return lm.handleSearchKey(msg)
		}
		switch {
		case lm.keymap.Up.matches(msg):
			if lm.cursor > 0 {
				lm.cursor--
			}
		case lm.keymap.Down.matches(msg):
			if lm.cursor < len(lm.issues)-1 {
				lm.cursor++
			}
		case lm.keymap.Home.matches(msg):
			lm.cursor = 0
		case lm.keymap.End.matches(msg):

// applyFetched stores the latest issue list and resets the cursor if
// it would otherwise point past the new list end.
func (lm listModel) applyFetched(msg tea.Msg) listModel {
	switch m := msg.(type) {
	case initialFetchMsg:
		lm.loading = false

			lm.issues = m.issues
		}
	}
	if lm.cursor >= len(lm.issues) {
		lm.cursor = max(0, len(lm.issues)-1)
	}
	return lm
```

```go
import (
	"fmt"
	"strings"

	"github.com/charmbracelet/lipgloss"
	"github.com/charmbracelet/lipgloss/table"
	"github.com/mattn/go-runewidth"

func (lm listModel) View(width, height int) string {
	if lm.loading {
		return statusStyle.Render("loading…")
	}
	if lm.err != nil {
		return errorStyle.Render(lm.err.Error())
	}
	if len(lm.issues) == 0 {
		return statusStyle.Render("no issues match. press c to clear filters or n to create one.")
	}
	rows := make([][]string, 0, len(lm.issues))
	for i, iss := range lm.issues {
		rows = append(rows, []string{
			selMarker(i == lm.cursor),
			fmt.Sprintf("#%d", iss.Number),
			statusChip(iss),
			truncate(iss.Title, max(20, width-50)),
			truncate(iss.Owner, 12),
			humanizeRelative(iss.UpdatedAt),
		})
	}
	t := table.New().
		Border(lipgloss.HiddenBorder()).
		Width(width).
		Wrap(false).
		Rows(rows...).
		StyleFunc(func(row, col int) lipgloss.Style {
			s := lipgloss.NewStyle()
			if row >= 0 && row < len(rows) && row == lm.cursor {
				s = s.Inherit(selectedStyle)
			}
			return s
		})
	return t.Render()
}

// helpers
func selMarker(selected bool) string {
	if selected {
		return "›"
	}
	return " "
}

func statusChip(iss Issue) string {
	switch {
	case iss.Deleted:
		return deletedStyle.Render("[deleted]")
	case iss.Status == "closed":
		return closedStyle.Render("closed")
	default:
		return openStyle.Render("open")
	}
}

func truncate(s string, w int) string {
	if w <= 0 || runewidth.StringWidth(s) <= w {
		return s
	}
	return runewidth.Truncate(s, w-1, "…")
}

// humanizeRelative converts an RFC3339 timestamp to "2d ago" style. For
// now stub it out; replace with a tiny helper. Keep ≤20 lines.

	if len(rfc3339) >= 10 {
		return rfc3339[:10]
	}
```

```go
type scope struct {
	projectID    int64  // 0 when allProjects or empty
	allProjects  bool
	empty        bool   // §7.2 empty state
	projectName  string
	workspace    string
}

func bootResolveScope(ctx context.Context, c *Client, allProjects bool, cwd string) (scope, error) {
	if allProjects {
		return scope{allProjects: true}, nil
	}
	rr, err := c.ResolveProject(ctx, cwd)
	if err == nil {
		return scope{projectID: rr.Project.ID, projectName: rr.Project.Name, workspace: rr.WorkspaceRoot}, nil
	}
	var apiErr *APIError
	if !errors.As(err, &apiErr) || apiErr.Code != "project_not_initialized" {
		return scope{}, err
	}
	// Soft fallback per §7.2.
	projects, listErr := c.ListProjects(ctx)
	if listErr != nil {
		return scope{}, listErr
	}
	if len(projects) == 0 {
		return scope{empty: true}, nil
	}
	return scope{allProjects: true}, nil
}
```

```go
package tui

import (
	"fmt"
	"os"
	"os/exec"

	tea "github.com/charmbracelet/bubbletea"

// editorCmd suspends Bubble Tea, runs $EDITOR on a tmpfile pre-seeded
// with `template`, and returns editorReturnedMsg with the final
// content. kind is one of "create" | "edit" | "comment" so the caller
// knows which mutation to dispatch.
func editorCmd(kind, template string) tea.Cmd {
	tmp, err := os.CreateTemp("", "kata-*.md")
	if err != nil {
		return func() tea.Msg { return editorReturnedMsg{kind: kind, err: err} }
	}
	if _, err := tmp.WriteString(template); err != nil {
		_ = tmp.Close()
		_ = os.Remove(tmp.Name())
		return func() tea.Msg { return editorReturnedMsg{kind: kind, err: err} }
	}
	_ = tmp.Close()
	editor := resolveEditor()
	cmd := exec.Command(editor[0], append(editor[1:], tmp.Name())...)
	return tea.ExecProcess(cmd, func(execErr error) tea.Msg {
		defer func() { _ = os.Remove(tmp.Name()) }()
		if execErr != nil {
			return editorReturnedMsg{kind: kind, err: execErr}
		}
		content, err := os.ReadFile(tmp.Name())
		if err != nil {
			return editorReturnedMsg{kind: kind, err: err}
		}
		return editorReturnedMsg{kind: kind, content: string(content)}
	})
}

// resolveEditor returns the command to invoke. Honors $EDITOR (with

	}
	return []string{"vi"}
}
```

```go
func startSSE(ctx context.Context, c *Client, projectID int64, sseCh chan<- sseMsg) {
	const maxBackoff = 30 * time.Second
	backoff := time.Second
	lastID := int64(0)
	for {
		select {
		case <-ctx.Done():
			return
		default:
		}
		if connected, err := readSSEStream(ctx, c, projectID, lastID, sseCh, &lastID); err != nil {
			notifyStatus(sseCh, sseReconnecting)
			if connected {
				backoff = time.Second
			}
			select {
			case <-ctx.Done():
				return
			case <-time.After(backoff):
			}
			backoff = min(backoff*2, maxBackoff)
			continue
		}
		return
	}
}
```

`readSSEStream` issues `GET /api/v1/events/stream?project_id=N` with `Last-Event-ID: <lastID>` (or `?after_id=<lastID>` if `lastID > 0`), parses frames, and emits `sseMsg` values for each frame.
